Installing Personal Navigator Skill

Personal Navigator works as a portable agent skill: a folder with `SKILL.md`, reference files, and memory templates. Full navigation begins after the 9-block Personality Map is built.

Main rule: the internal skill logic is written in English, but user-facing answers, interviews, the Map, journal, open loops, and ready-to-save memory blocks are created in the user's language.

What Gets Installed

personal-navigator-skill/
  SKILL.md
  agents/openai.yaml
  references/
  templates/

Codex

mkdir -p ~/.agents/skills
cp -R personal-navigator-skill ~/.agents/skills/

Restart Codex after installation and start with:

Use $personal-navigator-skill.
I want to start building my Personality Map. Guide me through a friendly interview.

Cross-Agent Workspace Path

For environments that support the open `SKILL.md` format, use:

mkdir -p .agents/skills
cp -R personal-navigator-skill .agents/skills/

Antigravity

Workspace: `.agents/skills/`. For global CLI installs, check the current Antigravity documentation.

Claude Code

Project: `.claude/skills/`. Personal: `~/.claude/skills/`.

OpenCode

Project: `.opencode/skills/`. Global: `~/.config/opencode/skills/`.

Windsurf / Cascade

Workspace: `.windsurf/skills/`. Global: `~/.codeium/windsurf/skills/`.

GitHub Copilot / VS Code

Project skills can live in `.github/skills/`, `.claude/skills/`, or `.agents/skills/`.

Cursor, Gemini CLI, Aider

Use `AGENTS.md` as a bridge: tell the agent to read `personal-navigator-skill/SKILL.md` and the relevant references.

Verify Installation

Use Personal Navigator.
I do not have a Personality Map yet. Start a friendly interview with me and create memory files if you can.

Expected behavior: the agent says full navigation is unavailable without a Map, creates or offers the 4 memory files, asks the first human question, and avoids deep conclusions without data.
